## Pillar Page Architecture (Your Content Hubs)

Each pillar page serves as a comprehensive hub that:
- Defines and positions a core concept (definitional authority)
- Links to 5-8 related cluster pages
- Establishes the ICP-specific angle HITL brings
- Becomes the "go-to resource" on that topic

### Pillar Pages (8 total):

1. **What is Workforce Architecture (and why it matters)?** → `/workforce-architecture/`
   - Positions HITL's foundational framework
   - Audience: Founders who don't know they need this

2. **How Do AI-Native Companies Structure Their Teams?** → `/ai-native-org-design/`
   - Positions the AI-first org design approach
   - Audience: COOs and ops leaders competing with AI-native peers

3. **What is a Super IC?** → `/super-ic/`
   - Defines the archetype, differentiates from senior engineer
   - Audience: All 3 ICPs (universal relevance)

4. **How Do I Decide: Hire vs. Automate vs. Super IC?** → `/hire-automate-or-super-ic/`
   - THE decision framework HITL teaches
   - Audience: All 3 ICPs (core pain point #3)

5. **How to Scale from Founder-Led to Structured Org** → `/post-funding-hiring/`
   - The Series A→B transition framework
   - Audience: Series B/C CEOs post-PMF

6. **How to Stop Hiring Reactively** → `/startup-recruiting/`
   - Hiring playbook and recruitment framework
   - Audience: Seed/Series A founders (core pain point #2)

## Internal Linking Strategy

### Pillar → Cluster
Each pillar page should link to its 4-6 supporting cluster pages:
```
/insights/workforce-architecture/
  ↓ (internal links to)
  /insights/workforce-architecture/what-roles-create-most-leverage/
  /insights/workforce-architecture/how-many-people-needed/
  /insights/workforce-architecture/hire-generalist-or-specialist/
  [etc.]
```

### Cross-Cluster Linking (Multiply Authority)
Related pages across different subpaths should link to each other:
```
/insights/ai-native-org-design/how-should-i-structure-teams/
  ↓ (internally links to)
  /insights/workforce-architecture/what-roles-create-most-leverage/
  /insights/hire-automate-or-super-ic/when-should-automate-vs-hire/
  /insights/super-ic/what-is-super-ic/
```

### ICP Variants (4x Authority)
When a question has variants for multiple ICPs, create internal cross-links:
```
Example: "When should I hire a manager?"
  - /post-funding-hiring/when-hire-first-manager/ (Generic)
  - /post-funding-hiring/when-should-seed-founder-hire-manager/ (ICP-1 variant)
  - /post-funding-hiring/when-series-b-ceo-hire-second-manager/ (ICP-2 variant)
  - /post-funding-hiring/when-ops-leader-build-team/ (ICP-3 variant)

All link to each other, signaling to crawlers that HITL owns this topic across contexts
```
